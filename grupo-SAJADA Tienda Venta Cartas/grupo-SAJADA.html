Tienda de venta cartas (Nombre por definir)

Entregable Tarea 1

"Hemos decidido como grupo desafiarnos a crear tienda de venta cartas (Nombre por definir) y buscamos en la medida de lo posible y lo comprendido en clase, crear algo cotidiano pero que tenga cierto nivel de complejidad a la hora de programarlo con tal de tener algo que podamos llamar como un buen desafio para un tema de proyecto."

La idea del proyecto es crear un sitio donde se puedan vender cartas de Magic, Pokemon o Yu-gioh. La página debera contar con un sistema de autenticación para el usuario vendedor en la cual se permita hacer cambios al catálogo de cartas, ver las ordenes con su respectivo estado, crear nuevos productos, modificar y eliminar existentes. Además la página principal en donde el usuario final tendrá acceso a consultar cartas por nombre especifico, por expansión o por juego. Como un desafío extra se tiene contemplado acceder a una API (Scryfall) designada en donde se consultaran los artes de cada carta y sus precios actuales, esta información será almacenada en nuestra base de datos con el fin de tenerla a mano y se mantendran los precios más actualizados.

Tecnologías a usar:

Se planea crear un front end basado en REACT y backend con . Además el motor de base de datos a utilizar será SQL con mysql.
